# cleansyst-dashboard

Дашборд (`index.html`) + PHP-прокси к API ThroneBaron (`proxy.php`).

## Деплой через GitHub Actions (без ручного FTP)

Деплой полностью автоматический. FTP-клиент открывать больше не нужно.

При каждом пуше в ветку `main` (а также при ручном запуске из вкладки **Actions**)
workflow [`.github/workflows/deploy.yml`](.github/workflows/deploy.yml) сам заливает
файлы на хостинг по FTP. Заливаются только изменённые файлы — это быстро.

### Настройка (делается один раз)

1. Откройте репозиторий на GitHub → **Settings** → **Secrets and variables** → **Actions**.
2. На вкладке **Secrets** нажмите **New repository secret** и добавьте три секрета:

   | Имя секрета    | Значение                                              |
   | -------------- | ----------------------------------------------------- |
   | `FTP_SERVER`   | адрес FTP-сервера, например `ftp.example.com`         |
   | `FTP_USERNAME` | логин FTP                                             |
   | `FTP_PASSWORD` | пароль FTP                                            |

3. (Необязательно) На вкладке **Variables** можно добавить переменные, если нужно
   изменить настройки по умолчанию:

   | Имя переменной    | Назначение                          | Значение по умолчанию |
   | ----------------- | ----------------------------------- | --------------------- |
   | `FTP_SERVER_DIR`  | каталог на сервере для заливки       | `./` (корень)         |
   | `FTP_PROTOCOL`    | протокол: `ftp`, `ftps`, `ftps-legacy` | `ftp`              |
   | `FTP_PORT`        | порт FTP-сервера                     | `21`                  |

### Как запустить деплой

- **Автоматически** — просто запушьте изменения в ветку `main`.
- **Вручную** — вкладка **Actions** → workflow **Deploy via FTP** → кнопка **Run workflow**.

### Что заливается

Заливаются все файлы репозитория, кроме служебных:
`.git`, `.github` и `README.md` (см. секцию `exclude` в workflow).
